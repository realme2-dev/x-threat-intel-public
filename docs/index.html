1. 현재 주요 사이버보안 트렌드


최근 공격자들은 공급망 공격(Supply Chain Attack)을 통해 개발자 도구(VS Code Extension 등)를 오염시켜 내부 리포지토리를 탈취하는 정교한 방식을 사용하고 있습니다. 또한, AI를 활용한 취약점 탐색 및 코드 분석이 가속화되면서 제로데이(Zero-day) 취약점의 발견과 악용 주기가 매우 짧아지고 있습니다. 특히, 인포스틸러(Infostealer)와 랜섬웨어(Ransomware)가 결합된 복합형 악성코드가 증가하고 있으며, 클라우드 환경의 자격 증명(Credential) 노출이 주요 공격 경로로 고착화되는 추세입니다.

2. 주요 위협 행위자 / 캠페인


  • TeamPCP: GitHub 내부 리포지토리 4,000여 개를 탈취한 공격 주체로 확인됨.

  • UNC6671: BlackFile 캠페인을 통해 Vishing(음성 피싱)과 결합된 대규모 갈취 활동 수행.

  • UNC6692: 사회 공학적 기법(Social Engineering)을 활용하여 맞춤형 악성코드 제품군을 배포.

  • Webworm: ESET에서 분석한 새로운 burrowing 기법을 사용하는 APT 그룹.

  • MoscowTeam_Steal: 스틸러, RAT, 랜섬웨어 기능을 통합한 복합 위협 악성코드 유포 주체.


    • 3. 한국 관련 위협


  • Nova: 대한민국 화장품 및 생활화학 기업을 대상으로 랜섬웨어 공격 수행 (AhnLab ASEC).

  • 피싱 메일: 국세청 및 하나은행을 정교하게 사칭한 PhaaS(Phishing-as-a-Service) 기반의 기업 타겟형 피싱 공격 확인 (이스트시큐리티).

  • 텔레그램 스미싱: 텔레그램 보안 문제를 사칭하여 계정 탈취를 유도하는 스미싱 공격 재확인 (AhnLab ASEC).


    • 4. 주요 CVE / 취약점


  • CVE-2026-45829: ChromaDB, CVSS 10.0, 악성 Hugging Face 모델 참조를 통한 Pre-auth RCE.

  • CVE-2026-9082: Drupal, Highly Critical, 인증 없이 정보 노출, 권한 상승 및 RCE 가능.

  • CVE-2026-46333: Linux Kernel, CVSS 5.5, 9년 된 취약점으로 로컬 Root 권한 획득 가능.

  • CVE-2026-41091 / CVE-2026-45498: Microsoft Defender, Zero-day, 권한 상승 및 DoS 악용 확인.

  • CVE-2026-3102: ExifTool, macOS 시스템 타겟, 악성 이미지를 통한 시스템 침해.

    6. 즉각 대응 권고


  • 공급망 보안 강화: 개발 환경에서 사용하는 VS Code 확장 프로그램의 무결성을 검증하고, 불필요한 확장 프로그램 사용을 제한하십시오.

  • 취약점 패치: Microsoft Defender 및 Linux Kernel(CVE-2026-46333), Drupal(CVE-2026-9082) 관련 보안 패치를 즉시 적용하십시오.

  • 자격 증명 보호: AWS Access Key 등 중요 자격 증명이 GitHub 등 공개 리포지토리에 노출되지 않았는지 전수 조사하고, MFA를 필수 적용하십시오.
