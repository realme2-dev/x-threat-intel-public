1. 현재 주요 사이버보안 트렌드


최근 위협 환경은 LLM 에이전트를 활용한 공격 자동화와 소프트웨어 공급망 공격이 결합된 형태로 진화하고 있습니다. 특히 공격자가 LLM을 통해 취약점 탐색부터 익스플로잇 코드 생성, 데이터 탈취까지 전 과정을 자동화하는 사례가 보고되고 있으며, 오픈소스 패키지 및 AI 모델 서버를 겨냥한 'Phantom Squatting'이 새로운 공급망 위협 벡터로 부상했습니다. 또한, 랜섬웨어 그룹은 데이터 암호화 없이 탈취만으로 협박하는 방식을 선호하며, 인프라 장비의 취약점을 악용하여 대규모 프록시 네트워크를 구축하는 등 공격의 정교함이 극대화되고 있습니다.

2. 주요 위협 행위자 / 캠페인


  • Kairos: 정부 기관을 대상으로 데이터를 암호화하지 않고 탈취 후 공개를 빌미로 100만 달러 규모의 금전을 갈취하는 Extortion 캠페인 수행.

  • North Korean Hackers (Contagious Interview 연계): npm, Packagist, Go, Chrome 확장 프로그램 등 108개의 악성 패키지를 배포하는 공급망 공격 수행.

  • JadePuffer: LLM 에이전트를 활용하여 랜섬웨어 공격 전 과정을 자동화한 최초의 사례로 보고됨.

  • Armored Likho: BusySnake Stealer를 사용하여 러시아 내 조직을 표적으로 스피어 피싱 및 AI 생성 로더를 활용한 공격 수행.

  • Scattered Spider: 영국 Transport for London 공격 등 대규모 인프라 타겟팅, 최근 주요 조직원 체포 및 유죄 판결 사례 발생.


    • 3. 한국 관련 위협


  • 북한발 공급망 공격: 이스트시큐리티 ESRC 보고에 따르면, 유명 테스트 프레임워크 Chai.js 플러그인으로 위장한 악성 npm 패키지(chai-as-init)가 국내 개발 환경을 겨냥해 배포됨.

  • 데이터 유출: ASEC 보고서에서 신규 데이터 강탈 그룹인 Settra가 한국 산업장비 제조 대기업의 해외 법인 데이터 유출을 주장함.

  • 기타: 일본 정부의 중국산 IoT/태양광 장비 전면 금지 조치 등 인접 국가의 보안 정책 변화가 국내 공급망 보안 기준에도 시사점을 줌.


    • 4. 주요 CVE / 취약점


  • CVE-2025-3248: MinIO, CVSS 9.8, 기본 자격 증명 악용 및 에이전트 기반 공격에 활용됨.

  • CVE-2026-29146: Apache Tomcat, Padding Oracle 취약점, 클러스터 메시지 위조 가능.

  • CVE-2026-7482: Ollama, 인증되지 않은 서버 메모리 덤프(AWS 키 포함) 가능.

  • CVE-2026-56141: JetBrains Hub, 인증 우회 및 빌드 파이프라인 RCE 유발.

  • CVE-2026-46242: Linux Kernel(epoll), UAF 취약점을 통한 권한 상승(Android 타겟).

  • 도메인: faith2dxy.xyz (CyberWarship), github.com/J-jaeyoung/bad-epoll (__su888)

  • CVE: CVE-2026-29146 (kmkz_security), CVE-2026-7482 (geeknik), CVE-2026-56141 (boss_sec_labo)


    • 6. 즉각 대응 권고


  • 공급망 점검: 개발 환경에서 사용하는 npm, Go 패키지 중 출처가 불분명하거나 최근 업데이트된 패키지에 대한 무결성 검증 및 chai-as-init 등 악성 패키지 포함 여부 확인.

  • AI/LLM 서버 보안: Ollama 등 내부 AI 서버가 인터넷에 노출되어 있는지 확인하고, 인증(Authentication) 설정 및 API 접근 제어 정책 즉시 적용.

  • 자격 증명 관리: MinIO, Tomcat 등 주요 인프라 솔루션의 기본 계정(default credentials) 사용 여부를 전수 조사하고 강력한 비밀번호로 교체.


    • 7. 위협 수준 평가
