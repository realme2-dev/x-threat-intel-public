🟡 위협 점수: 5/10 (보통)

1. 현재 주요 사이버보안 트렌드


최근 사이버 위협 환경은 WordPress Core의 RCE 취약점인 wp2shell과 같이 인증되지 않은 공격자가 원격 코드 실행을 수행할 수 있는 치명적인 취약점들이 공개되어 즉각적인 패치가 요구되는 상황입니다(BleepingComputer). 또한, 소프트웨어 공급망 공격이 지속적으로 발생하고 있으며, 특히 npm 패키지를 악용한 RAT 유포나 7-Zip의 RCE 취약점 등 신뢰받는 소프트웨어의 취약점을 노리는 공격이 두드러집니다(TheHackerNews, BleepingComputer). AI 기술이 보안 분석의 효율성을 높이는 동시에, 공격자들 또한 이를 활용해 공격 기법을 고도화하거나 자동화된 인증 우회 공격을 수행하는 등 AI 기반의 보안 공방이 격화되고 있습니다(이스트시큐리티 알약).

2. 주요 위협 행위자 / 캠페인


- Cavern Manticore: 이란 연계 APT 그룹으로, 모듈형 C2 프레임워크를 사용하여 주로 이스라엘을 타겟으로 활동함(Check Point Research).
- Scattered Spider: 영국에서 주요 리더들이 체포 및 66개월 징역형을 선고받았으며, 다수의 대규모 사이버 공격을 주도한 것으로 확인됨(CyberScoop).
- 북한 연계 공격자: chai-as-init 등 npm 악성 패키지를 배포하여 공급망 공격을 시도함(이스트시큐리티 알약).
- GoSerpent: 동남아시아 정부 기관을 타겟으로 Stowaway RAT, ThumbcacheService 등을 사용하여 데이터를 탈취하는 지속적인 위협 행위자(Securelist).

3. 한국 관련 위협


- 정부 정책: 2026년 하반기 업무계획을 통해 AI 데이터센터 보안 및 기반시설 취약점 점검 강화, AI 연구소 보안 요건 마련 등을 추진 중(데일리시큐).
- 한국어 트윗: 2018년 당시 발생한 CVE-2018-10561, CVE-2018-10562와 선거 인프라 보안 이슈를 언급하며, 현재의 사이버 위협 환경과 비교 분석하는 내용이 공유됨(Twitter).

4. 주요 CVE / 취약점


- CVE-2026-63030 / CVE-2026-60137: WordPress Core RCE 취약점, 공개 PoC 존재, 즉시 패치 필요(BleepingComputer).
- CVE-2026-56164: SharePoint 서버 대상 RCE 취약점, 현재 활발히 악용 중(Twitter).
- CVE-2026-46242: Linux Kernel epoll UAF 취약점, 로컬 권한 상승 가능(Twitter).
- CVE-2008-4128: 18년 된 취약점이나 최근 러시아 FSB 연계 활동에서 악용되어 CISA KEV에 추가됨(Twitter).

5. IOC 요약


- 해시: chai-as-init (npm 패키지 버전 v1.4.5~v1.4.7) (이스트시큐리티 알약)
- CVE: CVE-2026-63030, CVE-2026-60137, CVE-2026-56164, CVE-2026-46242, CVE-2008-4128 ([각 출처 참조])

6. 즉각 대응 권고


- WordPress 패치: wp2shell 취약점(CVE-2026-63030, CVE-2026-60137)에 대응하여 즉시 최신 버전으로 업데이트하십시오.
- SharePoint 보안 점검: CVE-2026-56164 악용 사례가 보고되었으므로 인터넷 노출된 SharePoint 서버의 패치 상태를 즉시 확인하십시오.
- 공급망 보안 강화: npm 패키지 사용 시 출처를 검증하고, chai-as-init와 같은 악성 패키지 포함 여부를 정기적으로 스캔하십시오.
