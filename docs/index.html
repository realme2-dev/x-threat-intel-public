1. 현재 주요 사이버보안 트렌드


최근 사이버 위협은 AI 모델의 취약점을 직접 타겟팅하거나, AI 에이전트 및 공급망을 악용하여 인증 토큰을 탈취하는 정교한 형태로 진화하고 있습니다. 특히 Gravity SMTP와 같은 대중적인 플러그인 취약점 악용과 더불어, EDR을 무력화하는 전용 프레임워크를 사용하는 RaaS 그룹의 활동이 두드러집니다. 또한, MITRE ATT&CK 프레임워크를 기반으로 한 방어 체계 고도화와 함께, AI 모델의 보안성을 확보하기 위한 '주권형 AI 보안(Sovereign AI Security)' 전략이 기업의 핵심 과제로 부상하고 있습니다.

2. 주요 위협 행위자 / 캠페인


  • The Gentlemen (Krebs on Security, WeLiveSecurity): GentleKiller EDR 프레임워크를 사용하여 400개 이상의 보안 프로세스를 무력화하는 공격 수행.

  • Sapphire Sleet (BlueNoroff) (BleepingComputer): Mastra AI 공급망 공격을 통해 140개 이상의 npm 패키지를 감염시킨 북한 연계 해킹 그룹.

  • Icarus (BleepingComputer): Klue의 OAuth 토큰을 탈취하여 고객사의 Salesforce 환경에 무단 접근.

  • TeamPCP (CyberScoop): 오픈소스 소프트웨어 생태계를 타겟으로 하여 보안보다 속도를 우선시하는 개발 환경의 허점을 악용.

  • Kimsuky (이스트시큐리티): 보안 담당자를 대상으로 개인정보 유출을 사칭한 스피어 피싱 공격 수행.


    • 3. 한국 관련 위협


  • 북한발 공급망 공격: chai-as-init 악성 npm 패키지가 Chai.js 플러그인으로 위장하여 배포됨 (이스트시큐리티).

  • 금융권 위협: 2026년 5월 기준, 국내 금융권 대상 피싱 공격이 2.3으로 급증하며 초기 침투의 주요 경로로 활용됨 (AhnLab ASEC).

  • 스피어 피싱: 보안 담당자를 대상으로 한 김수키(Kimsuky)의 정교한 스피어 피싱 공격 지속 (이스트시큐리티).

    6. 즉각 대응 권고


  • Gravity SMTP 패치: 사용 중인 WordPress 환경에서 Gravity SMTP 플러그인을 최신 버전으로 즉시 업데이트하고 API Key 유출 여부를 점검하십시오.

  • OAuth 및 서비스 계정 감사: Klue 사례와 같이 OAuth 토큰을 사용하는 연동 인프라의 권한을 재검토하고, 불필요한 레거시 자격 증명을 폐기하십시오.

  • 공급망 보안 강화: npm 등 오픈소스 패키지 사용 시 의존성 패키지의 무결성을 검증하고, chai-as-init과 같은 악성 패키지 포함 여부를 스캔하십시오.
